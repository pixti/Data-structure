# 11. 미로 문제 (Maze Problem)

미로 탐색은 입구에서 출구까지의 경로를 찾는 고전적인 알고리즘 문제입니다.   
막다른 길에 도달했을 때 가장 최근의 분기점으로 되돌아가는 **백트래킹(Backtracking)** 기법의 정수를 보여주며,   
이를 위해 **스택(Stack)** 자료구조를 필수적으로 사용합니다.

## 11-1. 미로의 수리적 표현 및 경계 설정

미로는 컴퓨터가 효율적으로 연산할 수 있도록 2차원 배열과 수치 데이터로 정의합니다.

### 1) 2차원 배열 활용
* **정의**: 미로는 $1 \le i \le m$, $1 \le j \le p$ 범위를 가진 2차원 배열 $maze[m][p]$로 표현합니다.
* **데이터 값의 의미**: 
    * **`1`**: 통로가 막혀 있음 (Wall)
    * **`0`**: 통과할 수 있음 (Path)
* **탐색 목표**: 입구 $maze[1][1]$에서 출발하여 출구 $maze[m][p]$까지의 경로 존재 여부 확인 및 좌표 추출.

### 2) 테두리 패딩 (Border Padding): 속도 최적화의 핵심
미로의 가장자리(행/열이 0 또는 끝인 지점)에 있을 때는 탐색 가능한 방향이 8개가 아니라 3~5개로 제한됩니다.  
이를 매번 `if`문으로 체크하면 코드가 복잡해지고 성능이 저하됩니다.  

* **해결책**: 미로의 주위를 모두 벽(`1`)으로 한 겹 더 둘러쌉니다.  
* **배열 선언**: **`maze[m+2][p+2]`**로 확장 선언합니다.  
* **장점**:   
    * 행이나 열이 0인지 매번 검사할 필요가 없어 **조건문 비교가 획기적으로 줄어듭니다.**  
    * 메모리는 아주 약간 더 사용하지만, **실행 속도가 매우 빨라지는** 영리한 프로그래밍 기법입니다.  

## 11-2. 이동 방향 정의: 8방향 이동 테이블 (Move Table)  

현재 위치 $(i, j)$에서 이동할 수 있는 8가지 방향을 좌표 변화량(Offset)으로 미리 정의하여 루프문으로 처리합니다.  

> **[이동 공식]**:   
> $nextRow = curRow + move[q].x$  
> $nextCol = curCol + move[q].y$  


**[방향 오프셋 테이블: `move[q]`]**  

| 방향 (q) | 약어 | move[q].x | move[q].y | 좌표 변화 설명 |  
| :--- | :---: | :---: | :---: | :--- |  
| **0** | **N** | -1 | 0 | 북쪽 (한 행 위로) |
| **1** | **NE** | -1 | 1 | 북동쪽 (우측 상단 대각선) |
| **2** | **E** | 0 | 1 | 동쪽 (한 열 오른쪽으로) |
| **3** | **SE** | 1 | 1 | 남동쪽 (우측 하단 대각선) |
| **4** | **S** | 1 | 0 | 남쪽 (한 행 아래로) |
| **5** | **SW** | 1 | -1 | 남서쪽 (좌측 하단 대각선) |
| **6** | **W** | 0 | -1 | 서쪽 (한 열 왼쪽으로) |
| **7** | **NW** | -1 | -1 | 북서쪽 (좌측 상단 대각선) |


## 11-3. 스택을 활용한 백트래킹 로직

스택은 **"내가 지나온 경로"** 와 **"해당 지점에서 다음에 시도해야 할 방향"** 을 기억하는 메모리 역할을 수행합니다.

### 1) 데이터 구조 (Element)  
스택의 한 칸에는 좌표뿐만 아니라 탐색의 연속성을 위해 다음 정보를 저장합니다.  
* `row`, `col`: 현재 위치의 좌표  
* `dir`: 이 지점에서 다음에 탐색해야 할 방향 번호 (0~7)  

### 2) 무한 루프 방지: 방문 기록 (Marking)  
* **문제**: 서쪽에서 왔는데 다시 서쪽을 탐색하면 왔던 길을 되돌아가며 무한 루프에 빠집니다.  
* **해결**: 방문한 좌표를 `mark[row][col] = 1`로 표시하거나 `maze` 값을 수정하여 재방문을 금지합니다.  

### 3) 탐색 상세 과정 (Logic Flow)  
1. **시작**: 입구 $(1, 1)$과 시작 방향을 스택에 `push`합니다.    
2. **Push (전진)**: 
    - 현재 위치에서 이동 가능한 방향(`maze == 0` && `mark == 0`)을 찾으면,  
    - 이동하기 전 **현재 좌표와 다음에 시도할 방향 번호(q+1)**  를 스택에 넣습니다. (실패 시 이 지점부터 재시작하기 위함)  
3. **Pop (백트래킹)**:  
    - 8방향을 모두 조사했는데 갈 곳이 없다면(막다른 길), 스택에서 하나를 꺼냅니다(`Pop`).
    - 꺼낸 위치(직전 갈림길)로 되돌아가서, 저장해두었던 **다음 방향 번호**부터 다시 탐색을 재개합니다.  
4. **종료**: 출구 $(m, p)$ 에 도달하면 성공, 스택이 빌 때까지 못 찾으면 경로 없음으로 종료합니다.


## 11-4. 알고리즘 분석 및 코드 구조

### 1) 성능 분석 
| 항목 | 상세 내용 |  
| :--- | :--- |  
| **핵심 자료구조** | **스택 (Stack)**: LIFO 특성을 이용해 가장 최근의 분기점으로 복귀 |
| **공간 복잡도** |  $O(m \times p)$ : 미로 크기만큼의 배열 공간 및 최악의 경우 스택 공간 필요 |
| **시간 복잡도** |  $O(m \times p)$ : 최악의 경우 모든 통로를 한 번씩만 방문 |

### 2) 로직
```java
// 8방향 탐색 루프  
for (int i = 0; i < MAX_MOVE; i++) {  
    nextRow = curRow + move[i].x;  
    nextCol = curCol + move[i].y;  

    // 이동 가능 조건: 벽(1)이 아니고, 방문(mark)한 적 없음  
    if (maze[nextRow][nextCol] == 0 && mark[nextRow][nextCol] == 0) {  
        // 현재 좌표와 '다음 시도 방향(i+1)'을 저장 후 전진  
        push(curRow, curCol, i + 1);   
        
        curRow = nextRow;   
        curCol = nextCol;  
        mark[nextRow][nextCol] = 1; // 방문 표시  
        i = -1; // 새 위치에서 다시 0번(N) 방향부터 탐색하기 위해 초기화  
    }  
}
