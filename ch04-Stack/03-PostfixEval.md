# 10. 수식의 표기법 (Expression Notation)

수식 표기법은 **연산자(Operator)** 와 **피연산자(Operand)** 의 배치 순서에 따라 구분됩니다.   
컴파일러가 인간의 언어(중위)를 기계적 계산(후위)으로 변환하는 과정에서 **스택(Stack)** 자료구조가 핵심적인 역할을 수행합니다.

* **핵심 원칙**: 어떤 표기법을 사용하더라도 **피연산자의 상대적 순서는 절대 변하지 않습니다.** 오직 연산자의 위치만 바뀝니다.

## 10-1. 표기법의 종류 및 특징

### 1) 중위 표기법 (Infix Notation)
* **정의**: 연산자를 두 피연산자 **사이**에 위치시키는 방법 (예: $A + B$ )
* **특징**: 인간이 가장 직관적으로 이해할 수 있는 방식입니다.
* **단점**: 연산 우선순위(곱셈, 나눗셈 등)를 제어하기 위해 반드시 **괄호**가 필요하며,  
  컴퓨터가 한 번에 읽어서 계산하기 복잡합니다.

### 2) 후위 표기법 (Postfix Notation) / 역폴란드 표기법
* **정의**: 연산자를 피연산자 **뒤** 에 위치시키는 방법 (예: $AB+$ )
* **특징**: **컴파일러**가 수식을 계산할 때 주로 사용합니다.
* **장점**: 괄호가 전혀 필요 없으며, 수식을 왼쪽에서 오른쪽으로 스캔하며 스택을 이용해 즉시 계산이 가능합니다.

### 3) 전위 표기법 (Prefix Notation) / 폴란드 표기법
* **정의**: 연산자를 피연산자 **앞** 에 위치시키는 방법 (예: $+AB$ )
* **특징**: 연산자가 먼저 등장하여 뒤에 올 피연산자들의 연산을 미리 예고하는 형태입니다.

## 10-2. 표기법 변환 비교표

| 중위 표기법 (Infix) | 후위 표기법 (Postfix) | 전위 표기법 (Prefix) |
| :--- | :--- | :--- |
| $A + B$ | $AB+$ | $+AB$ |
| $A + B - C$ | $AB+C-$ | $-+ABC$ |
| $A + B * C - D$ | $ABC*+D-$ | $-+A*BCD$ |
| $(A + B) / (C - D)$ | $AB+CD-/$ | $/+AB-CD$ |

## 10-3.  후위 표기법 수식 계산 로직

컴퓨터가 후위 표기법을 계산할 때는 스택을 활용하여 단 한 번의 스캔( $O(N)$ )으로 결과를 도출합니다.

>  **[로직]** : 피연산자는 스택에 **Push**하고, 연산자를 만나면 2회 **Pop**하여 계산한 뒤 결과를 다시 **Push** 합니다.

**[계산 상세 과정]**
1. **피연산자(Operand)** 를 읽으면: 스택에 `push`합니다.
2. **연산자(Operator)** 를 읽으면: 
   * 스택에서 두 번 `pop`을 수행합니다. (먼저 나온 값이 뒤쪽 피연산자, 나중에 나온 값이 앞쪽 피연산자가 됨)
   * `(나중에 pop된 값) [연산자] (먼저 pop된 값)`을 수행합니다.
   * 계산된 결과값을 다시 스택에 `push`합니다.
3. **종료**: 입력을 모두 읽은 후 스택에 남은 최종값이 수식의 결과입니다.


## 10-4. 중위표기법  → 후위표기법 변환 로직
컴파일러가 인간의 수식을 기계용으로 바꾸는 알고리즘입니다.

**[변환 규칙]**
1. **피연산자**: 읽는 즉시 출력(Output)합니다.
2. **연산자**: 스택 `top`에 있는 연산자와 우선순위를 비교합니다.
   * 스택 `top`의 우선순위가 **나보다 낮을 때까지** `pop`하여 출력하고, 자신을 `push`합니다.
   * (즉, 스택 내부에는 위로 갈수록 우선순위가 높은 연산자가 쌓여야 함)
3. **괄호 처리**:
   * **왼쪽 괄호 `(`**: 우선순위가 가장 높은 것으로 간주하여 무조건 `push`합니다.
   * **오른쪽 괄호 `)`**: 왼쪽 괄호 `(`가 나올 때까지 스택의 모든 연산자를 `pop`하여 출력합니다. (괄호 자체는 출력하지 않음)
4. **마무리**: 입력을 다 읽으면 스택에 남은 모든 연산자를 `pop`하여 출력합니다.

## 10-6. 컴파일러의 괄호 쌍 맞추기 (괄호 검사)
수식의 유효성을 검사할 때도 스택이 사용됩니다.

* **Push 조건**: 왼쪽 괄호 `(`, `{`, `[`를 읽으면 스택에 넣습니다.
* **Pop 조건**: 오른쪽 괄호 `)`, `}`, `]`를 읽으면 스택에서 `pop`하여 짝이 맞는지 확인합니다.
* **에러 상황**:
  1. 오른쪽 괄호를 읽었는데 스택이 비어 있는 경우 (과잉 닫힘)
  2. `pop`된 왼쪽 괄호와 읽은 오른쪽 괄호의 종류가 다른 경우 (짝 불일치)
  3. 모든 수식을 읽었는데 스택에 괄호가 남아 있는 경우 (과잉 열림)

## 10-7. 연산자 우선순위의 효율적 관리: ISP & ICP

단순히 `if-else`나 `while`문에 모든 연산자 케이스를 나열하는 방식은 코드의 가독성과 유지보수성이 매우 떨어집니다. 
이를 해결하기 위해 **ISP(In-Stack Precedence)** 와 **ICP(In-Coming Precedence)** 개념을 도입합니다.

* **배경**: 괄호 `(`는 스택 **밖**에 있을 때는 무조건 들어가야 하므로 높은 우선순위를 갖지만,  
  스택 **안**에 들어간 후에는 다른 연산자들에 의해 쉽게 `pop`되지 않도록 낮은 우선순위를 가져야 합니다.  
  이 이중성을 수치로 해결합니다.

### 1) ISP와 ICP를 이용한 스택 관리
* **ISP (In-Stack Priority)**: 연산자가 스택 **내부**에 있을 때의 우선순위
* **ICP (In-Coming Priority)**: 연산자가 스택 **외부**에서 진입할 때의 우선순위

> **[로직]**: 연산자는 자신의 **isp**가 새로운 연산자의 **icp**보다 **작거나 같을 때**  
> 스택에서 `pop`되어 출력됩니다. (우선순위 숫자가 작을수록 먼저 처리됨)

### 2) 우선순위 수치 테이블 (Standard Guide)

| 연산자 | ISP (내부) | ICP (외부) | 비고 |
| :--- | :---: | :---: | :--- |
| **`(`** | **8** | **0** | 진입 시 0순위, 진입 후엔 8순위로 고정 |
| **단항 (`-`, `!`)** | 1 | 1 | |
| **산술 (`*`, `/`, `%`)** | 2 | 2 | |
| **산술 (`+`, `-`)** | 3 | 3 | |
| **비교 (`<`, `<=`, `>=`, `>`)** | 4 | 4 | |
| **비교 (`==`, `!=`)** | 5 | 5 | |
| **논리 (`&&`)** | 6 | 6 | |
| **논리 (`\|\|`)** | 7 | 7 | |
| **`#(eos)`** | 8 | 8 | 수식의 끝 |


## 10-8. 다중 자릿수 및 공백 처리

컴퓨터는 문자를 하나씩 읽기 때문에 '253'과 같은 숫자를 '2', '5', '3'이라는 개별 문자로 인식합니다. 이를 하나의 숫자로 합치는 로직이 필요합니다.

### 1) 다중 자릿수 숫자 병합 (Multi-digit Merge)
현재 읽은 문자가 숫자일 때, **바로 앞의 문자도 숫자였다면** 이를 하나의 연속된 숫자로 판단하여 합산 연산을 수행합니다.

* **연산 공식**:
  $$\text{NewValue} = (\text{StackTop} \times 10) + \text{CurrentDigit}$$
* **예시**: 스택 `top`에 `25`가 있고 현재 읽은 문자가 `'3'`인 경우
  1. 스택에서 `25`를 `pop()` 합니다.
  2. $25 \times 10 + 3 = 253$을 계산합니다.
  3. 결과값 `253`을 다시 스택에 `push()` 합니다.

### 2) 공백(space) 처리
* **역할**: 숫자와 숫자 사이를 구분하거나 수식의 가독성을 높입니다.
* **로직**: 수식을 읽는 도중 공백을 만나면 아무런 연산 없이 **무시하고 다음 문자로** 넘어갑니다.


## 10-9. 최종 수식 계산 알고리즘 요약

1. **숫자(Digit)인 경우**: 
   - `push()` 하되, 연속된 숫자라면 위 공식을 통해 기존 `top`과 합친 후 저장합니다.
2. **연산자(Operator)인 경우**: 
   - 스택에서 피연산자 2개를 `pop()` 하여 연산하고 결과를 다시 `push()` 합니다.
3. **공백(Space)인 경우**: 
   - 무시하고 다음 토큰으로 진행합니다.
