## 1. 스택 (Stack)

스택은 **LIFO (Last-In First-Out, 후입선출)** 라는 엄격한 규칙을 가진 선형 자료구조입니다.   
가장 나중에 들어온 데이터가 가장 먼저 나가는 '쌓기' 형태의 구조를 가집니다.

## 1-1. 스택도 추상 자료형(ADT)이다.
스택은 내부가 배열인지 연결 리스트인지에 상관없이 **"LIFO 규칙만 만족하면 스택"** 입니다.

* **ADT (명세)**: "데이터를 쌓고(Push), 맨 위에서 꺼낸다(Pop)"는 기능을 정의한 설계도.
* **자료구조 (구현)**: 설계도를 바탕으로 실제 배열이나 연결 리스트를 사용하여 물리적으로 구현한 실체.

## 1-2. 주요 연산 (Stack ADT)

| 연산 | 설명 |
| :--- | :--- |
| **push(item)** | 스택의 가장 위에 새로운 항목을 추가함 |
| **pop()** | 가장 위에 있는 항목을 제거하고 그 값을 반환함 |
| **top() / peek()** | 가장 위에 있는 항목을 제거하지 않고 값만 읽어옴 |
| **isEmpty()** | 스택이 비어있는지 확인 (비었으면 true) |
| **popAll()** | 스택의 모든 항목을 한 번에 제거함 |

 스택은 '검색(Search)' 기능을 설계하지 않습니다.   
 오직 **"지금 내 눈앞(Top)에 있는 데이터"** 에만 집중하는 것이 스택의 존재 목적이기 때문입니다.


## 2. 스택의 시간 복잡도

스택의 모든 연산은 **입구(Top)** 한 곳에서만 일어나기 때문에 매우 효율적입니다.

| 연산 유형 | 시간 복잡도 | 설명 |
| :--- | :---: | :--- |
| **Push / Pop / Top** | **$O(1)$** | 데이터 개수와 상관없이 즉시 수행 |
| **탐색 (Search)** | $O(N)$ | 최악의 경우 다 꺼내봐야 함 (스택답지 않은 연산) |
| **접근 (Access)** | $O(N)$ | 중간 데이터를 꺼내려면 위의 데이터를 다 치워야 함 |


## 3. 배열 기반 스택 (ArrayStack)

배열을 저장 공간으로 사용하는 방식입니다.

### ① `topIndex`의 역할
배열 스택의 핵심 엔진입니다. 현재 스택의 '맨 위' 위치를 기억합니다.
* **초기 상태 (`topIndex = -1`)**: 0번 인덱스에 첫 데이터를 넣기 위해 -1에서 시작하는 것이 가장 일반적입니다.

### ② 두 가지 `popAll()`과 메모리 누수(Memory Leak)
단순히 인덱스만 초기화하는 방식은 Java에서 위험할 수 있습니다.

1. **인덱스 리셋 (`topIndex = -1`)**:
   - **논리적**으로는 비어있지만, **물리적**으로는 배열이 객체들의 주소값을 여전히 붙잡고 있습니다.
   - 가비지 컬렉터(GC)가 해당 객체들을 지우지 못해 **메모리 누수**가 발생합니다.
2. **새 배열 할당 방식 (`stack = new Object[size]`)**:
   - 낡은 배열을 버리고 깨끗한 새 배열을 연결합니다.
   - 참조를 잃은 낡은 배열과 객체들은 GC가 수거해 갑니다.

## 4. 연결 리스트 기반 스택 (LinkedListStack)

배열의 '크기 제한'을 해결하기 위해 노드(Node)를 사용하는 방식입니다.

### 4-1. 왜 Top은 무조건 '헤드(Head)'여야 하는가?
단방향 연결 리스트의 특성 때문에 입구(Top)의 위치 선정이 성능을 결정합니다.

* **Top = Head (권장)**: 맨 앞에 넣고 빼는 것은 항상 **$O(1)$** 입니다.
* **Top = Tail (비권장)**: 삽입은 빠를지 몰라도, **삭제(Pop)** 할 때 `tail`의 이전 노드를 찾기 위해 처음부터 다 훑어야 하므로 **$O(N)$**의 시간이 발생합니다.

### 4-2. 주요 로직
* **Push**: 새 노드의 `next`를 기존 `topNode`에 연결하고, `topNode`를 새 노드로 갱신합니다. (배열과 달리 `isFull` 체크가 필요 없다.)
* **Pop**: 현재 `topNode`의 데이터를 챙긴 뒤, `topNode`를 `topNode.next`로 한 칸 옮깁니다.
* **PopAll**: `topNode = null` 한 줄이면 끝납니다. 헤드만 끊기면 나머지 노드들은 GC에 소거됩니다.


> **[최종 요약]**
> * **ArrayStack**: 조회가 빠르고 구현이 쉽지만, 크기가 고정적이고 메모리 관리(GC)에 신경 써야 한다.
> * **LinkedListStack**: 크기가 무한 확장 가능하고 삽입/삭제가 깔끔하지만, 노드를 생성할 때마다 추가적인 메모리 공간이 필요하다.
