## 10. AVL 트리의 삭제 연산  
AVL 트리의 삭제는 단순히 노드를 제거하는 것에 그치지 않고,  
루트 방향으로 거슬러 올라가며 발생할 수 있는 '연쇄적 불균형'을 모두 해결해야 합니다.   

### 10-1. 삭제 연산의 2단계 프로세스   
AVL 트리에서의 삭제는 다음 두 단계를 거쳐 진행됩니다.   

1.  **[1단계] BST 방식의 삭제 수행**:     
    * 이진 탐색 트리(BST)의 삭제 규칙에 따라 노드를 제거합니다.   
    * 자식이 없는 경우, 자식이 하나인 경우, 자식이 둘인 경우(후계자 이용)로 나누어 처리합니다.   
2.  **[2단계] 루트 방향으로 균형 복구**:   
    * 실제 노드가 삭제된 위치의 부모 노드부터 루트 노드 방향으로 거슬러 올라갑니다.   
    * 각 노드에서 균형 인수($BF$)를 확인하고, 불균형이 발생($|BF| \ge 2$)하면 적절한 **회전 연산**을 수행합니다.   
    * **주의**: 삽입과 달리 삭제는 회전 후에도 상위 노드에서 다시 불균형이 발생할 수 있어, 루트까지 계속 점검해야 합니다.    

### 10-2. 핵심 아이디어: "반대쪽 삽입"으로 간주  
삭제 후 불균형이 발생했을 때, 어떤 회전을 할지 결정하는 기준은 다음과 같습니다.   

<img width="1484" height="411" alt="image" src="https://github.com/user-attachments/assets/cb5fb711-b42e-4e04-8315-074cf5a01881" />  


> **아이디어**: 특정 서브트리에서 노드가 삭제된 것은,
> **반대쪽 서브트리에 노드가 삽입되어 불균형이 발생한 것**과 논리적으로 동일하게 취급합니다.    

* **상황 설정**: 삭제된 노드의 부모를 $p$, $p$의 부모를 $gp$, $p$의 형제 노드를 $s$라고 할 때:  
    * $s$의 왼쪽 서브트리가 높으면 $\rightarrow$ **L 유형**  
    * $s$의 오른쪽 서브트리가 높으면 $\rightarrow$ **R 유형**  
* 이를 통해 **LL, RR, LR, RL** 중 적절한 회전을 선택합니다.  


### 10-3. 삭제 연산의 회전 유형
삭제된 노드의 반대쪽(형제 노드 $s$ 쪽)의 모양에 따라 결정됩니다.  

| 불균형 상태 (삭제 후) | 대응하는 회전 연산 | 비고 |  
| :--- | :--- | :--- |  
| **왼쪽 서브트리 비대 (L)** | **LL** 또는 **LR** 회전 | 형제의 자식 중 높은 쪽 기준 |  
| **오른쪽 서브트리 비대 (R)** | **RR** 또는 **RL** 회전 | 형제의 자식 중 높은 쪽 기준 |   

### 10-4. 예제 (40 삭제)  
노드 **40**을 삭제하는 복합적인 과정을 분석합니다.  

1.  **삭제 전 상태**: 루트 50, 왼쪽 서브트리에 30, 10, 20, 40 등이 배치된 상태.
2.  **40 삭제**: 40은 자식이 없는 단말 노드이므로 단순히 제거됩니다. 

<img width="504" height="464" alt="image" src="https://github.com/user-attachments/assets/34d79321-ee4f-4dc4-b981-59906121fc4a" />  
  
3.  **1차 불균형 발견 (노드 30)**:  
    * 40이 삭제되면서 30의 오른쪽 자식이 `null`이 됩니다.  
    * $BF(30) = h(10, 20) - h(null) = 2 - 0 = 2$ (**불균형 발생**).    
    * 왼쪽 자식(10)의 오른쪽 서브트리(20)가 더 높으므로 **LR 불균형**으로 판단.   

<img width="504" height="397" alt="image" src="https://github.com/user-attachments/assets/8cf85e14-9bc7-4c06-bc50-64e694e73155" />  
  
4.  **1차 회전 (LR 회전)**:   
    * 10에 대해 좌회전 후 30에 대해 우회전 수행.   
    * **결과**: 20이 위로 올라오고 10과 30이 자식이 됨.   
  
<img width="470" height="442" alt="image" src="https://github.com/user-attachments/assets/754c10d1-6ce8-4dc3-a9d0-310c5e43fd67" />  

5.  **2차 불균형 발견 (노드 50)**:  
    * 아래에서 회전이 일어났음에도 불구하고, 루트 50 기준 왼쪽 높이가 줄어들어 다시 불균형 발생.  
    * 오른쪽 서브트리(75) 쪽이 더 높으므로 **RL 불균형** 상황.  
6.  **2차 회전 (RL 회전)**:  
    * 75의 왼쪽 자식인 60을 중심으로 회전 수행.  
    * **결과**: 60이 새로운 루트가 되며 트리 전체가 완벽한 균형을 되찾음.

### 10-5. 삭제 시간 복잡도    
* **수행 시간**: $O(\log N)$  
* **이유**: 삭제 대상을 찾는 탐색에 $\log N$, 삭제 후 루트까지 올라가며 균형을 체크하고 회전하는 과정에 $\log N$이 소요됩니다.  
  회전 자체는 $O(1)$이므로 전체 복잡도는 로그 시간을 유지합니다.

## 11. AVL 트리의 수행 시간 (Running Time) 종합 분석
AVL 트리는 일반적인 이진 탐색 트리(BST)의 단점을 보완하여,   
어떤 데이터가 들어오더라도 최악의 상황을 방지하고 일정한 성능을 보장하도록 설계된 자료구조입니다.  

### 11-1. 연산별 수행 시간 원리  
AVL 트리의 모든 핵심 연산(탐색, 삽입, 삭제)은 트리의 **높이($h$)** 에 직접적으로 비례합니다.  

* **탐색 (Search)**: 루트 노드부터 시작하여 하향(Descending)하며 목표 노드 또는 이파리 노드까지 내려갑니다.  
  비교 횟수는 트리의 높이를 넘지 않습니다.  
* **삽입 및 삭제 (Insert & Delete)**:  
    * **하향 단계**: 삽입/삭제 위치를 찾기 위해 루트부터 내려갑니다. ( $O(h)$ )  
    * **상향 단계 (Backtracking)**: 연산 후 다시 루트 노드 방향으로 거슬러 올라가며 불균형을 체크하고 높이를 갱신합니다. ( $O(h)$ )  
    * **회전 연산**: 불균형 발견 시 수행되는 회전은 포인터 몇 개만 바꾸는 작업이므로 각 층마다 $O(1)$ 시간밖에 걸리지 않습니다.  

---

### 11-2. BST vs AVL 수행 시간 비교  
일반 BST와 AVL 트리의 가장 큰 차이는 **최악의 경우(Worst Case)** 에 대한 보장 여부입니다.  

| 연산 | 일반 BST (평균) | 일반 BST (최악) | AVL 트리 (평균/최악) |  
| :--- | :--- | :--- | :--- |  
| **탐색 (Search)** | $O(\log N)$ | $O(N)$ | **$O(\log N)$** |  
| **삽입 (Insert)** | $O(\log N)$ | $O(N)$ | **$O(\log N)$** |  
| **삭제 (Delete)** | $O(\log N)$ | $O(N)$ | **$O(\log N)$** |  

* **BST의 한계**: 데이터가 정렬된 순서(1, 2, 3...)로 삽입되면 트리가 한쪽으로 치우쳐 성능이 $O(N)$으로 급격히 저하됩니다.  
* **AVL의 강점**: 스스로 균형을 잡기 때문에 높이가 항상 $O(\log N)$ 으로 유지되어, 어떤 상황에서도 로그 시간 복잡도를 보장합니다.  

### 11-3. 데이터 특성에 따른 성능 최적화  
다양한 실험 결과에 따르면 각 트리가 최적의 성능을 발휘하는 시나리오가 다릅니다.  

* **AVL 트리**: **거의 정렬된 데이터**를 삽입한 후 랜덤 순서로 데이터를 탐색하는 경우 가장 좋은 성능을 보입니다.  
  (정렬된 데이터를 넣어도 스스로 균형을 맞춰 높이를 최소화하기 때문)  
* **일반 이진 탐색 트리**: **랜덤 순서의 데이터**를 삽입한 후에 랜덤 순서로 데이터를 탐색하는 경우 가장 좋은 성능을 보입니다.  
  (무작위 삽입 자체가 어느 정도의 자연적인 균형을 만들어주기 때문)  

### 11-4. 수학적 결론  
노드 수가 $N$개일 때, AVL 트리는 다음의 성능을 수학적으로 보장합니다.  

* **높이 제약**: $h < 1.44 \log_2(N+2) - 1.328 \approx O(\log N)$  
* **최종 복잡도**: 탐색, 삽입, 삭제 모두 **$O(\log N)$**  

> AVL 트리는 삽입과 삭제 시 '회전'이라는 추가적인 비용( $O(1)$ )을 지불하지만, 이를 통해 트리의 높이를 항상 로그 수준으로 묶어둡니다.   
> 덕분에 데이터의 입력 순서에 상관없이 일관되게 빠른 속도를 보장받을 수 있는 매우 안정적인 자료구조입니다.
