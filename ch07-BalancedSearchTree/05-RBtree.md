# 12. 레드-블랙 트리 (Red-Black Tree)  
레드-블랙 트리는 자가 균형 이진 탐색 트리로, 노드에 색상을 부여하여 트리의 균형을 유지합니다.  

## 12-1. 레드-블랙 트리의 정의 및 기본 특성  

<img width="703" height="504" alt="image" src="https://github.com/user-attachments/assets/3c9f2b80-5145-4ab7-a77b-ebaa581e0a60" />  


### 1) 레드-블랙 트리 (RB Tree)의 구조  
* **리프 노드의 재정의**: 모든 `null` 자리에 가상의 **리프 노드(External Node)** 를 둡니다.
  레드-블랙 트리에서 말하는 '리프 노드'는 바로 이 `null` 리프를 의미합니다.  
* **노드 색상**: 트리의 모든 노드는 **레드(Red)** 또는 **블랙(Black)** 중 하나의 색상을 갖습니다.  

### 2) 레드-블랙 트리의 3대 필수 특성  
트리가 "Red-Black"으로 인정받기 위해서는 다음 규칙을 반드시 만족해야 합니다.  
1.  **루트와 리프의 색상**: 루트 노드와 모든 리프 노드(`null`)는 반드시 **블랙**이어야 합니다.  
2.  **레드 노드의 연속 금지**: 루트로부터 임의의 리프 노드에 이르는 경로 상에  
    **레드 노드 두 개가 연속으로 출현할 수 없습니다.** (즉, 레드 노드의 자식은 항상 블랙입니다.)  
3.  **블랙 높이(Black Height)의 일치**: 루트 노드에서 임의의 리프 노드에 이르는 모든 경로에서 만나는 **블랙 노드의 수(Black Height)는 모두 같습니다.**  

## 12-2. 확장 이진 트리 및 포인터 관점의 성질  
일반적인 노드 기준의 성질을 확장하여, 노드 간의 **포인터(Edge)** 에 색상을 부여하는 관점으로도 해석할 수 있습니다.  

<img width="1075" height="399" alt="image" src="https://github.com/user-attachments/assets/67c01908-df59-4210-87d0-60803b0b2013" />  


### 1) 노드 컬러링 관점 (RB1 ~ RB3)  
* **RB1**: 루트와 모든 외부 노드(External Node)들은 컬러가 블랙이다.  
* **RB2**: 루트에서 외부 노드로 이르는 경로는 두 개의 연속적인 레드 노드를 가질 수 없다.  
* **RB3**: 루트에서 외부 노드까지의 모든 경로 상에 있는 블랙 노드의 수는 동일하다.  

### 2) 포인터 컬러링 관점 (RB1' ~ RB3')  
노드의 색상을 부모-자식 간 포인터의 색상으로 치환하여 생각할 때의 성질입니다.  
* **RB1'**: 내부 노드로부터 외부 노드로 가는 마지막 포인터는 블랙이다.  
* **RB2'**: 루트에서 외부 노드로 이르는 경로는 두 개의 연속적인 레드 포인터를 가질 수 없다.  
* **RB3'**: 루트에서 외부 노드로의 모든 경로에 있는 블랙 포인터의 수는 동일하다.  

## 12-3. 수학적 보조 정리 및 증명  
레드-블랙 트리가 왜 $O(\log N)$ 의 성능을 보장하는지 수학적으로 분석한 내용입니다.  

### [보조 정리 10.1] 경로 길이의 균형  
> 루트로부터 외부 노드로 이르는 임의의 두 경로 $P, Q$ 가 있을 때,   
> $$length(P) \le 2 \times length(Q)$$ 가 성립한다.  

**[증명]**  
1.  임의의 레드-블랙 트리에서 루트의 블랙 랭크(Rank)를 $r$이라고 가정합니다.  
2.  **RB1'** 에 의해 루트에서 외부 노드로 가는 경로의 마지막 포인터는 항상 블랙입니다.  
3.  **RB2'** 에 의해 레드 포인터는 연속해서 나타날 수 없으므로, **각 레드 포인터 뒤에는 반드시 블랙 포인터**가 와야 합니다.  
4.  따라서 전체 경로에서 레드 포인터의 수는 블랙 포인터의 수보다 많을 수 없습니다.  
5.  결과적으로 루트에서 외부 노드까지의 각 경로는 최소 $r$개(전부 블랙일 때)에서 최대 $2r$개(블랙-레드 교차 시) 사이의 포인터를 갖게 됩니다.  
6.  그러므로 어떤 경로라도 다른 경로보다 2배를 초과하여 길어질 수 없습니다. $\therefore length(P) \le 2 \times length(Q)$  

### [보조 정리 10.2] 트리의 높이와 노드 수의 관계  
> 레드-블랙 트리의 높이를 $h$, 내부 노드 수를 $n$, 블랙 랭크를 $r$이라 하면 다음이 성립한다.  
> (a) $h \le 2r$    
> (b) $n \ge 2^r - 1$    
> (c) $h \le 2\log_2(n+1)$  

**[증명]**  
* **(a) 증명**: 보조 정리 10.1에 의해 경로의 최대 길이는 $2r$을 넘지 않으므로, 트리의 높이 $h \le 2r$입니다.  
* **(b) 증명**: 레벨 1에서 $r$까지는 외부 노드가 존재하지 않습니다.   
  블랙 높이가 $r$인 이진 트리에서 외부 노드가 나타나기 전까지 채워지는 최소한의 내부 노드 수는 포화 이진 트리의 형태를 띠므로 $2^r - 1$개입니다.  
  따라서 $n \ge 2^r - 1$입니다.  
* **(c) 증명**: (b)의 식을 $r$에 대해 정리하면:  
    $$n+1 \ge 2^r \implies \log_2(n+1) \ge r$$  
    이를 (a) 식인 $h \le 2r$에 대입하면:  
    $$h \le 2\log_2(n+1)$$ 이 결과는 트리의 높이 $h$가 노드 수 $n$에 대해 로그 함수에 비례함을 의미하며,  
    탐색 성능이 $O(\log N)$임을 수학적으로 완벽히 보장합니다.  

# 12-4. 레드-블랙 트리의 구현과 탐색  
### 1) 레드-블랙 트리의 표현 (Representation)  
* **외부 노드의 처리**: 실제 메모리를 차지하는 노드 객체 대신 **널 포인터(null pointer)** 를 사용하여 외부 리프 노드를 표현합니다.
  이는 메모리 사용량을 절감하는 효과가 있습니다.  
* **색상 저장 비용**:  
    * **노드 중심**: 노드 내부에 색상 정보를 저장할 경우, 0(Red)과 1(Black)을 구분할 **1bit**의 추가 공간만 필요합니다.  
    * **포인터 중심**: 자식으로 내려가는 포인터에 색상을 부여하는 구현 방식의 경우, 
      왼쪽/오른쪽 자식 각각의 색상을 저장하기 위해 노드당 **2bit**가 필요합니다.    
* **탐색 (Search)**: 일반적인 **이진 탐색 트리(BST)** 의 탐색 알고리즘을 그대로 사용합니다.
    균형이 보장되어 있으므로 시간 복잡도는 항상 $O(\log N)$ 입니다.  

# 12-5. 삽입 시 노드 컬러 지정 원칙
새로운 노드를 삽입할 때, 레드-블랙 트리는 다음과 같은 규칙을 따릅니다.  

1.  **루트 노드**: 삽입된 노드가 루트라면 무조건 **블랙(Black)** 으로 지정합니다.  
2.  **비루트 노드**: 새로 삽입되는 노드는 일단 **레드(Red)** 로 지정합니다.
    * **왜 레드로 삽입하나?**: 블랙으로 삽입하면 그 즉시 해당 경로의 **블랙 높이(Black Height)가 변하여**  
    * '모든 경로의 블랙 높이가 같다'는 규칙을 깨뜨립니다. 반면, 레드로 삽입하면 '레드 연속 금지' 규칙만 체크하면 되므로 불균형 조정이 상대적으로 수월합니다.  
3.  **불균형 조정**: 만약 부모 노드가 레드라면 **Double Red**가 발생하며, 이때 주변 노드의 색상에 따라 불균형을 조정합니다.  

# 12-6. 불균형 판단의 핵심 요소: $gu, pu, u$와 삼촌 노드  
불균형을 분석하기 위해 트리의 3대 가계도와 '삼촌'의 상태를 확인해야 합니다.  

* **$u$ (Current Node / New Node)**: 현재 문제를 일으키고 있는 **새로 삽입된 레드 노드**입니다.  
* **$pu$ (Parent Node)**: $u$의 부모 노드입니다. $pu$가 레드일 때 'Double Red' 문제가 발생합니다.  
* **$gu$ (Grandparent Node)**: $u$의 조부모 노드($pu$의 부모)입니다.  
* **삼촌(Uncle) 노드**: $gu$의 자식 중 $pu$가 아닌 나머지 한쪽 자식입니다.  
    * *예시*: $pu$가 $gu$의 왼쪽 자식이라면, $gu$의 오른쪽 자식이 삼촌이 됩니다.

# 12-7. 불균형 타입 명칭의 비밀: X, Y, r, b의 의미  
명칭(예: **LLr**, **LRb**)만 봐도 어떤 상황인지 즉시 파악할 수 있도록 약속된 기호입니다.  

### 1) 방향을 결정하는 X와 Y  
* **X (조부모와 부모의 관계)**: $pu$가 $gu$의 어느 쪽에 있는가?  
    * **L**: $pu$가 $gu$의 **왼쪽(Left)** 자식임.  
    * **R**: $pu$가 $gu$의 **오른쪽(Right)** 자식임.  
* **Y (부모와 나의 관계)**: $u$가 $pu$의 어느 쪽에 있는가?  
    * **L**: $u$가 $pu$의 **왼쪽(Left)** 자식임.  
    * **R**: $u$가 $pu$의 **오른쪽(Right)** 자식임.  

### 2) 해결 방식을 결정하는 r과 b   
* **r (Red Uncle)**: 삼촌 노드가 **레드(Red)** 인 경우 $\to$ **Recoloring(색상 변경)** 으로 해결.    
* **b (Black Uncle)**: 삼촌 노드가 **블랙(Black)** 인 경우 (NIL/null 포함) $\to$ **Rotation(회전)** 으로 해결.    


# 12-8. 삼촌이 레드일 때: XYr 타입 (Recoloring)  
삼촌이 레드라면 트리의 물리적 구조를 바꿀 필요 없이 색상만 조정하여 블랙 높이를 맞춥니다.  

### [해결 로직]  
1.  **부모($pu$)**와 **삼촌 노드**를 **블랙(Black)** 으로 바꾼다.  
2.  **조부모($gu$)**를 **레드(Red)** 로 바꾼다.  
  
### [특징 및 전파]    
* **장점**: 회전 연산이 없어 처리가 매우 빠릅니다.  
* **단점**: 조부모($gu$)가 레드가 되면서, **조부모의 부모와 다시 'Double Red'** 가 발생할 수 있습니다.  
  이 경우 조부모를 새로운 $u$로 설정하고 다시 루트 방향으로 올라가며 반복 검사합니다 (**Recursive Propagation**).  

| 타입 | 상황 설명 | 조치 |  
| :--- | :--- | :--- |  
| **LLr** | $pu$는 $gu$의 L, $u$는 $pu$의 L, 삼촌 Red | $pu$, 삼촌 $\to$ B / $gu \to$ R |  
| **LRr** | $pu$는 $gu$의 L, $u$는 $pu$ of R, 삼촌 Red | $pu$, 삼촌 $\to$ B / $gu \to$ R |  
| **RRr** | $pu$는 $gu$의 R, $u$는 $pu$ of R, 삼촌 Red | $pu$, 삼촌 $\to$ B / $gu \to$ R |  
| **RLr** | $pu$는 $gu$의 R, $u$는 $pu$ of L, 삼촌 Red | $pu$, 삼촌 $\to$ B / $gu \to$ R |  


# 12-9. 삼촌이 블랙일 때: XYb 타입 (Rotation)  
삼촌이 블랙이면 단순히 색상만 바꿔서는 블랙 높이 규칙을 지킬 수 없습니다. 물리적으로 트리를 회전시켜 높이를 맞춰야 합니다.  

### 1) LLb / RRb (단일 회전 타입)  
노드들이 한쪽 방향으로 곧게 뻗어 있는 경우입니다.  
* **LLb**: $gu$를 기준으로 **우회전(Right Rotate)** 수행.  
* **RRb**: $gu$를 기준으로 **좌회전(Left Rotate)** 수행.  
* **색상 조정**: 회전 후 중심이 된 노드($pu$)는 **블랙**, 그 자식들은 **레드**가 됨.  

### 2) LRb / RLb (이중 회전 타입)  
노드들이 꺾여 있는 경우입니다. 한 번에 해결되지 않아 두 번 회전합니다.  
  
* **LRb**: $pu$를 기준으로 좌회전($\to$ LL화) 후, $gu$를 기준으로 우회전 수행.  
* **RLb**: $pu$를 기준으로 우회전($\to$ RR화) 후, $gu$를 기준으로 좌회전 수행.  
* **색상 조정**: 최종적으로 꼭대기에 올라온 노드($u$)가 **블랙**, 양옆 자식들이 **레드**가 됨.  

### [XYb의 결정적 차이점]  
XYb 연산을 마치고 나면 해당 서브트리의 루트는 **블랙**이 됩니다. 
블랙 노드는 레드 노드와 충돌하지 않으므로, **더 이상 위쪽 트리로 불균형이 전파되지 않고 그 자리에서 즉시 종료**됩니다.
