## 1. 해시 테이블 (Hash Table)

## 1-1. 탄생 배경: 더 빠른 탐색을 향하여
기존에 데이터를 탐색하기 위해 사용하던 주요 자료구조들은 데이터의 양이 많아질수록 탐색 시간이 늘어나는 한계가 있었습니다.

* **Array, Linked List, BST (최악의 경우):** $O(N)$
* **AVL 트리, B-트리 등 (균형 탐색 트리):** $O(\log N)$

**해시 테이블(Hash Table)** 은 $O(\log N)$ 보다 더 빠른 연산을 위해 고안되었습니다. 디자인 철학에 맞게 잘 설계된다면 데이터 개수와 상관없이 아주 빠른 검색, 삽입, 삭제 작업을 **평균 $\Theta(1)$ (상수 시간)** 에 수행할 수 있는, 
이론적으로 색인 효율의 극한에 위치한 자료구조입니다.

## 1-2. 해시 테이블의 핵심 아이디어
키(Key)와 1차원 배열의 인덱스(Index) 사이의 관계를 이용하여 데이터를 저장합니다.

* 다른 킷값과의 상대적인 크기를 비교하여 위치를 정하는 것이 아니라, **자신의 킷값 자체에 의해 저장 위치가 즉시 결정**됩니다.
* **문제점:** 만약 넓은 범위의 키를 배열의 인덱스로 '그대로' 사용한다면, 데이터가 없는 빈 공간이 많아져 **메모리 낭비**가 심해집니다.
* **해결 방안:** 이 문제를 해결하기 위해 키를 적절한 크기의 인덱스로 변환하는 **해싱(Hashing)** 기법을 사용합니다.

## 1-3. 해싱(Hashing)의 주요 구성 요소

| 용어 | 설명 |
| :--- | :--- |
| **해싱 (Hashing)** | 키를 간단한 함수를 사용해 변환한 값을 배열의 인덱스로 이용하여 항목을 저장하는 전체적인 과정 |
| **해시 함수 (Hash Function)** | 해싱에 사용되는 함수. 키 값을 입력받아 저장할 위치를 계산해 냅니다. (예: h(k) = k%13) |
| **해시 값 (Hash Value)** | 해시 함수가 계산해 낸 결과값. (즉, 해시 테이블의 인덱스 주소) |
| **해시 테이블 (Hash Table)** | 항목이 해시 값에 따라 실제로 저장되는 1차원 배열 |

## 1-4. 좋은 해시 함수의 조건
가장 이상적인 해시 함수는 키들을 해시 테이블의 인덱스로 **균등하게(Uniformly)** 변환하는 함수입니다.

### ① 균등한 분배 (랜덤한 흩어짐)
* 학번, 주민번호처럼 이미 의미나 특성이 부여된 키의 가장 앞/뒤 몇 자리만 취해서 해시값으로 사용하면, 특정 인덱스에 데이터가 몰리는 **충돌(Collision)** 이 많이 발생합니다.
* 따라서 의미가 부여되어 있는 키를 간단한 계산을 통해 '뒤죽박죽'으로 만든 후, 해시 테이블 크기($M$)에 맞도록 계산하여 데이터를 랜덤하게 흩어지도록 해야 합니다.

### ② 빠른 계산 속도
* 아무리 키를 균등하게 잘 분배하는 해시 함수라도, 함수 계산 자체에 긴 시간이 소요된다면 해시 테이블의 가장 큰 장점인 **'연산의 신속성'** 을 상실하게 됩니다. 따라서 연산이 단순하고 빨라야 합니다.

## 1-5. 연산 및 시간 복잡도 요약

| 연산 유형 | 평균 시간 복잡도 | 비고 |
| :--- | :---: | :--- |
| **검색 (Search)** | $\Theta(1)$ | 킷값을 해시 함수에 넣어 바로 인덱스를 찾아냄 |
| **삽입 (Insert)** | $\Theta(1)$ | 계산된 인덱스 위치에 즉시 저장 |
| **삭제 (Delete)** | $\Theta(1)$ | 계산된 인덱스 위치의 데이터를 즉시 삭제 |

## 2. 대표적인 해시 함수 (Hash Functions)

앞서 살펴본 '균등한 분배'를 달성하기 위해 사용되는 대표적인 해시 함수들을 정리합니다.

### 2-1. 나눗셈 (Division) 함수
가장 직관적이고 널리 사용되는 기본적인 해시 함수입니다.

* **원리:** 키(key)를 해시 테이블의 크기인 $M$으로 나눈 **나머지**를 해시값으로 사용합니다.
* **수식:** $h(key) = key \mathbin{\%} M$ (결과값은 $0$부터 $M-1$ 사이의 인덱스)
* **특징 (소수 사용의 중요성):** $M$은 주로 **소수(Prime Number)** 를 사용합니다. 제수(나누는 수)로 소수를 사용할 때, 단순 나눗셈 연산만으로도 키들이 특정 인덱스에 몰리지 않고 테이블 전체에 균등하게 흩어지는 수학적 성질을 가지기 때문입니다.

### 2-2. 중간제곱 (Mid-square) 함수
* **원리:** 키 값을 통째로 **제곱**한 후, 그 결과값에서 적당한 크기의 **중간 부분**을 추출하여 해시값으로 사용합니다.
* **특징:** 키의 모든 자릿수가 제곱 연산에 참여하므로, 키의 일부분만 조금 달라도 최종 결과가 크게 달라져 랜덤성을 높이는 데 유리합니다.

### 2-3. 접기 (Folding) 함수
주로 큰 자릿수의 십진수 키(예: 주민번호, 전화번호 등)를 다룰 때 유용합니다.

* **원리:** 긴 키를 일정한 길이(몇 자리) 단위로 끊은 뒤, 나누어진 각 숫자들을 모두 **합산**하여 해시값을 만듭니다.
* **예시:** 키가 `123456789012`일 때, 4자리씩 끊어 더합니다.
  * $1234 + 5678 + 9012 = 15924$
  * 계산된 합계(`15924`)가 테이블 크기를 넘는다면, 이 값에 나눗셈 함수 등을 추가로 적용하여 최종 인덱스로 변환합니다.

### 2-4. 곱셈 (Multiplicative) 함수
나눗셈 함수와 달리 테이블 크기 $M$이 굳이 소수가 아니어도 비교적 균등하게 분배되는 방법입니다.  
h(key) = (int) (key * $\delta$ ) & 1 ) * M

* **원리:** $1$보다 작은 실수 $\delta$를 키에 곱한 뒤, 그 숫자의 **소수 부분**만 취합니다. 그 소수 부분에 다시 테이블 크기 $M$을 곱하고, **정수 부분**만 잘라내어 해시값으로 사용합니다.
* **Knuth의 제안:** 컴퓨터 과학자 도널드 크누스(Donald Knuth)에 의하면 $\delta = \frac{\sqrt{5}-1}{2} \approx 0.61803$ (황금비와 관련)일 때 가장 좋은 성능(균등한 분배)을 보인다고 합니다.
* **계산 예시:** (조건: $M = 127$, $key = 123456789$, $\delta = 0.61803$)
  1. **키와 실수 곱하기:** $123456789 \times 0.61803 = 76299999.30567$
  2. **소수 부분만 추출:** $0.30567$
  3. **소수 부분에 $M$ 곱하기:** $0.30567 \times 127 = 38.82009$
  4. **정수 부분만 추출:** 최종 해시값은 **$38$**

## 2-5. 좋은 해시 함수들의 공통점 요약

위에서 살펴본 함수들은 '효율적인 해싱'을 위해 다음과 같은 공통된 설계 철학을 가집니다.

1. **모든 자릿수 활용:** 키의 특정 앞/뒤 몇 자리만 쓰는 것이 아니라, 모든 자리의 숫자를 계산에 참여시킵니다.
2. **의미 소거 (Randomization):** 연산 과정을 거친 결과값은 원래 키가 가지고 있던 고유의 의미나 규칙성(예: 특정 연도, 특정 지역 번호)을 전혀 찾아볼 수 없는 '뒤죽박죽'인 상태가 됩니다.
3. **크기 최적화:** 무작위로 흩어진 결과값에서, 해시 테이블의 크기($M$)에 맞춰 특정 부분이나 나머지 연산을 활용해 안전하고 정확하게 인덱스로 매핑합니다.
