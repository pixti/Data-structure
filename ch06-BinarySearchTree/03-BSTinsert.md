## 5. 삽입 연산 (Insert)  
이진 탐색 트리에서 새로운 데이터를 추가하는 '삽입'은 탐색 연산의 논리를 그대로 활용하여 적절한 위치를 찾아가는 과정입니다.  

### 5-1. 삽입의 기본 원리 (Basic Logic) 
삽입 연산의 핵심은 **"탐색이 실패한 바로 그 지점"**  이 새로운 노드가 들어갈 자리라는 점입니다.

1.  **탐색 수행**: 삽입하려는 키 $x$를 사용하여 트리 탐색을 시작합니다.  
2.  **위치 탐색**: $x$가 현재 노드의 키보다 작으면 왼쪽, 크면 오른쪽으로 이동합니다.  
3.  **노드 생성 및 연결**: 탐색 중 `null`에 도달하면(탐색 실패 지점), 그 위치에 새로운 노드를 생성하고 부모 노드와 연결합니다.  

> **특징**: 이진 탐색 트리에서 새로운 노드의 삽입은 항상 **단말(Leaf) 노드** 수준에서 발생합니다.

### 5-2. 삽입 알고리즘: 재귀와 반복
삽입은 구현 방식에 따라 재귀적 방법과 비재귀적(반복문) 방법으로 나뉩니다.  

| 구분 | 재귀적 버전 (Recursive) | 비재귀적 버전 (Iterative) |  
| :--- | :--- | :--- |  
| **핵심 논리** | 서브트리의 루트를 반환하여 부모와 재연결 | `while`문을 통해 내려가며 부모 노드를 추적 |  
| **특징** | 코드가 간결하고 논리적 흐름이 명확함 | 스택 오버플로우 위험이 없으나 부모 노드 추적 로직 필요 |  
| **반환값** | 작업 완료 후 해당 (서브)트리의 루트 레퍼런스 | 보통 반환값 없음 (필드 직접 수정) |  


### 5-3. Java 구현 및 중복 키 처리  
실제 구현 시에는 `put(Key k, Value v)` 메소드를 주로 사용하며, 키의 중복 여부에 따라 동작이 달라집니다.   

* **중복 키가 없는 경우**: 탐색 실패 지점에 새 `Node(k, v)`를 생성하여 삽입합니다.   
* **이미 키가 존재하는 경우**: 새로운 노드를 만드는 대신, 해당 노드의 **Value(값)를 새로운 값으로 갱신(Update)** 합니다.   

#### 재연결(Re-linking) 메커니즘  
재귀적 구현에서 `n.setLeft(put(n.getLeft(), k, v))`와 같은 코드는 삽입 후 하위에서부터 루트까지 거슬러 올라가며    
노드들 사이의 링크를 다시 설정하는 역할을 합니다. 이는 트리의 무결성을 유지하는 안전한 방식입니다.   


### 5-4. 삽입 연산의 과정 시각화   
키 `35`를 삽입하는 과정을 예로 들면 다음과 같습니다.  

1.  `root(50)`보다 작으므로 **왼쪽** 이동.  
2.  `30`보다 크므로 **오른쪽** 이동.  
3.  `40`보다 작으므로 **왼쪽** 이동 시도.  
4.  왼쪽 자식이 `null`이므로 그 자리에 `35` 노드 생성 및 연결.

### 5-5. 시간 복잡도 (Time Complexity)   
삽입 연산의 효율성 역시 탐색과 마찬가지로 트리의 구조에 결정됩니다.   

* **시간 복잡도**: $O(h)$ (여기서 $h$는 트리의 높이)  
* **분석**: 최악의 경우 루트에서 단말 노드까지 한 번 내려가는 경로만큼 비교 연산을 수행하므로 트리의 높이($h$)에 비례하는 시간이 소요됩니다.  
* **성능 보장**: 균형 잡힌 트리에서는 $O(\log N)$을 유지하지만, 편향된 트리에서는 $O(N)$까지 성능이 저하될 수 있습니다.
