## 4. 이진 탐색 트리의 탐색(search) 연산  
이진 탐색 트리(BST)에서 특정 키를 가진 노드를 찾는 과정은 트리의 계층적 구조를 따라 내려가는 선택의 연속입니다.  

### 4-1. 탐색 연산의 기본 논리 (Basic Logic)  
탐색은 항상 **루트 노드** 에서 시작하며, 찾고자 하는 키 $k$와 현재 노드의 키 $id$를 비교하여 세 가지 방향 중 하나를 결정합니다.  

1.  **일치 ($k == id$)**: 탐색 성공. 해당 노드의 데이터를 반환합니다.  
2.  **작음 ($k < id$)**: 현재 노드보다 왼쪽 서브트리에 존재할 가능성이 높으므로 **왼쪽 자식**으로 이동합니다.  
3.  **큼 ($k > id$)**: 현재 노드보다 오른쪽 서브트리에 존재할 가능성이 높으므로 **오른쪽 자식**으로 이동합니다.  


### 4-2. 재귀적 탐색 메커니즘 (Recursive Mechanism)  
탐색 연산은 동일한 비교 로직이 하위 트리에서 반복되는 구조이므로, 재귀 알고리즘으로 명확하게 표현됩니다.  

> **알고리즘: TreeSearch(p, k)**  
> * **기본 케이스 (실패)**: $p$가 `null`인 경우, 트리 끝까지 도달했으나 키를 찾지 못한 것입니다.  
> * **기본 케이스 (성공)**: $k$와 $p.id$가 같으면 해당 노드를 반환합니다.  
> * **재귀 단계**:  
>   * $k < p.id$ 이면 `TreeSearch(p.left, k)`를 호출합니다.  
>   * $k > p.id$ 이면 `TreeSearch(p.right, k)`를 호출합니다.  

### 4-3. 탐색 성공과 실패

탐색의 결과는 경로의 끝이 어디냐에 따라 두 가지로 나뉩니다.    

* **성공적인 탐색 (Successful Search)**: 루트에서 시작하여 키 $x$를 포함하는 내부 노드에서 멈추는 경우입니다.   
* **실패한 탐색 (Unsuccessful Search)**: 키가 존재하지 않아 단말 노드의 자식인 외부(`null`) 영역까지 도달하는 경우입니다.   

| 구분 | 탐색 성공 | 탐색 실패 | 
| :--- | :--- | :--- |   
| **종료 지점** | 키가 일치하는 실제 노드 | `null` (External Node) |  
| **반환값** | 해당 노드의 Value | `null` |  
| **비교 횟수** | 해당 노드의 레벨 + 1 | 해당 경로의 전체 높이만큼 |  

### 4-4. 시간 복잡도 분석 (Complexity Analysis)
탐색 연산의 효율성은 트리의 '모양'에 의해 결정되는 **높이($h$)** 에 비례합니다.

* **시간 복잡도**: **$O(h)$**  
* **분석**:  
    * 각 레벨에서 비교 연산은 $O(1)$의 시간이 걸립니다.  
    * 루트에서 가장 깊은 곳까지 내려가더라도 최대 트리의 높이($h$)만큼의 노드만 방문하면 됩니다.   
* **보충 설명**:   
    * 트리가 균형 잡힌 경우(Balanced), 높이 $h = \log_2 N$이 되어 매우 효율적입니다.  
    * 트리가 한쪽으로 치우친 경우(Skewed), 높이 $h = N$이 되어 선형 탐색과 다를 바 없는 성능을 보입니다.  
> 탐색 성능을 보장하기 위해 단순히 데이터를 넣는 것에 그치지 않고, 트리의 높이를 최소화하는 '균형 유지'가 중요한 이유입니다.
