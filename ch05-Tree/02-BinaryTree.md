## 2. 이진 트리 (Binary Tree)
이진 트리는 모든 노드가 최대 2개의 자식 노드만을 가질 수 있는 트리입니다.  
단순히 자식이 적은 것이 아니라, 왼쪽과 오른쪽의 위치 정보가 매우 중요하다는 특징이 있습니다.  

### 2-1. 이진 트리의 정의
이진 트리는 다음과 같이 **재귀적(Recursive)** 으로 정의됩니다.  
  
**Empty(공집합)이거나,**  
**루트 노드** 와 2개의 이진 트리인 **왼쪽 서브트리(Left Subtree),** **오른쪽 서브트리(Right Subtree)** 로 구성됩니다.

#### [주요 케이스] (모양이 다르면 다른 트리로 간주)  
(a) Empty 트리: 노드가 하나도 없는 상태도 이진 트리의 일종입니다.  
(b) 루트만 있는 트리: 자식이 모두 null인 경우입니다.  
(c) 왼쪽 자식만 있는 트리: 오른쪽 서브트리가 비어 있는 경우입니다.  
(d) 오른쪽 자식만 있는 트리: 왼쪽 서브트리가 비어 있는 경우입니다.  

### 2-2. 특별한 형태의 이진 트리
이진 트리가 노드를 얼마나 꽉 채우고 있느냐에 따라 아래와 같이 분류합니다.

#### ① 포화 이진 트리 (Full Binary Tree)
정의: **모든 내부 노드가 정확히 2개의 자식** 을 가지며, 모든 단말 노드가 같은 레벨에 있는 트리입니다.  
특징: **빈틈없이 꽉 차 있는 상태** 로, 레벨이 $h$일 때 노드의 총 개수는 항상 $2^h - 1$개입니다.

#### ② 완전 이진 트리 (Complete Binary Tree)  
정의: 마지막 레벨을 제외한 모든 레벨이 노드로 꽉 차 있고, **마지막 레벨은 왼쪽부터 차례대로 노드가 채워진 트리** 입니다.   
특징:   
포화 이진 트리는 항상 완전 이진 트리입니다.  
하지만 완전 이진 트리라고 해서 모두 포화 이진 트리인 것은 아닙니다.  
**배열** 로 구현할 때 인덱스 낭비 없이 가장 효율적으로 저장할 수 있는 형태입니다.  

#### ③ 편향 트리(Skewed Tree)
중간 레벨에 빈 공간이 있거나, 마지막 레벨이 왼쪽부터 채워지지 않은 경우.  
한쪽 방향으로만 노드가 쏠린 경우. (최악의 경우 탐색 속도가 연결리스트와 같은 $O(N)$ 이 됨)


## 2-3. 이진 트리의 속성 (Properties of Binary Tree)
이진 트리의 형태(포화, 완전 등)에 따라 노드 수와 트리의 높이($h$) 사이에는 수학적인 규칙성이 존재합니다.  

### ① 레벨 및 노드 수 관련 성질
* **레벨 $k$에 있는 최대 노드 수**: $2^{k-1}$ (단, $k = 1, 2, 3, \dots$)  
    - 한 층의 최대 노드 수는 이전 층 노드 수의 **2배**입니다. (각 노드가 최대 2개의 자식을 가지므로)  
* **높이가 $h$인 포화 이진 트리(Full Binary Tree)의 총 노드 수 ($n$):**
    - $2^0 + 2^1 + 2^2 + \dots + 2^{h-1} = 2^h - 1$
* **높이가 $h$인 포화 이진 트리의 리프 노드(Leaf Node) 수:** $2^{h-1}$


### ② 높이($h$)와 노드 수($n$)의 관계
임의의 이진 트리에서 노드 수 $n$과 높이 $h$는 다음의 범위를 가집니다.

1. **최소 노드 수**: 높이가 $h$일 때, 적어도 $h$개의 노드가 있어야 합니다. (편향 트리의 경우)  
2. **최대 노드 수**: 높이가 $h$일 때, 최대 $2^h - 1$개의 노드를 가질 수 있습니다. (포화 이진 트리의 경우)  
3. **높이의 범위**: $n$개의 노드를 가진 이진 트리의 높이 $h$는  
    $$\lceil \log_2(n+1) \rceil \le h \le n$$  
    - **최소 높이**: $\lceil \log_2(n+1) \rceil$ (완전 이진 트리 형태일 때)  
    - **최대 높이**: $n$ (경사/편향 트리 형태일 때)  


### ③ 이진 탐색 트리(BST)의 성능 (상태에 따른 차이)
이진 탐색 트리에서 삽입과 검색에 드는 시간은 **트리의 높이($h$)**에 비례합니다.  

* **평균적인 경우**: 노드가 골고루 퍼져 있다면(완전 이진 트리 형태), 검색 시간은 $\Theta(\log n)$ 입니다.  
* **최악의 경우**: 트리가 한쪽으로 치우친 편향 트리일 경우, 높이 $h$가 $n$과 같아져 검색 시간이 $O(n)$ 으로 늘어납니다.  
  (연결 리스트와 동일해짐)

### ④ 리프 노드($n_0$)와 차수가 2인 노드($n_2$)의 관계
모든 이진 트리에서 자식이 없는 리프 노드의 수($n_0$)와 자식이 2개인 노드의 수($n_2$) 사이에는 항상 다음의 공식이 성립합니다.  

$$n_0 = n_2 + 1$$  

(자식이 둘인 노드보다 리프 노드가 항상 1개 더 많음)  

#### [증명]
1. **노드 관점**: 총 노드 수 $n = n_0 + n_1 + n_2$ (차수가 0, 1, 2인 노드의 합)  
2. **가지(Edge) 관점**:   
    - 루트를 제외한 모든 노드는 부모로부터 오는 가지가 하나씩 있으므로, 총 가지 수 $B = n - 1$ 입니다.  
    - 또한, 모든 가지는 차수가 1인 노드에서 1개, 2인 노드에서 2개씩 뻗어 나오므로 $B = n_1 + 2n_2$ 입니다.  
3. **결합**: $n - 1 = n_1 + 2n_2 \implies (n_0 + n_1 + n_2) - 1 = n_1 + 2n_2$  
4. **결론**: 양변에서 $n_1, n_2$를 정리하면 **$n_0 = n_2 + 1$** 이 도출됩니다.  

### 2-4. 이진 트리의 구현 (Implementation)  
이진 트리는 데이터의 형태와 사용 목적에 따라 배열 또는 연결 리스트로 구현할 수 있습니다.  

**① 배열을 이용한 구현 (Static Implementation)**
주로 **완전 이진 트리(Complete Binary Tree)** 를 저장할 때 사용합니다. 노드에 번호를 매겨 배열의 인덱스로 활용합니다.

#### 1. 루트 인덱스가 1인 경우 (표준)
구현이 직관적이며 계산이 간편하여 가장 많이 쓰입니다. (0번 인덱스는 비워둠)  
(트리에 N개의 노드가 있다고 가정)  
  
arr[i]의 부모 노드: `arr[i / 2]` (단, i > 1)  
arr[i]의 왼쪽 자식: `arr[2 * i]` (단, 2 x i ≤ N)  
arr[i]의 오른쪽 자식: `arr[2 * i + 1]` (단, 2 x i + 1 ≤ N)  

#### 2. 루트 인덱스가 0인 경우
프로그래밍 언어의 배열 특성(0-based)을 그대로 따를 때 사용합니다.  

arr[i]의 부모 노드: `arr[ (i-1) / 2]`  
arr[i]의 왼쪽 자식: `arr[2 * i + 1]`   
arr[i]의 오른쪽 자식: `arr[2 * i + 2]`  

#### 3. 배열 방식의 특징
**장점**: 인덱스 계산만으로 부모/자식 노드를 즉시 찾을 수 있어 접근 속도가 매우 빠릅니다.   
**단점 (편향 이진 트리의 문제)**: 트리의 높이가 $h$ 일 때, 최대 $2^h - 1$ 크기의 배열이 필요합니다.   
편향(Skewed) 트리의 경우, 노드 수는 적은데 높이만 높아서 대부분의 배열 공간이 **null** 로 채워지는 심각한 메모리 낭비가 발생합니다.  

**② 연결 리스트를 이용한 구현 (Link-based / Recursive)**  
각 노드를 객체로 만들고 레퍼런스(포인터)로 연결하는 방식입니다. **일반적인 트리 구현의 표준** 입니다.   
   
```java
class Node {
    Object key;
    Node left;   // 왼쪽 자식 가리킴
    Node right;  // 오른쪽 자식 가리킴
}
```
#### 1. 연결 리스트 방식의 특징  
**장점:**   
트리의 모양에 상관없이 필요한 만큼만 노드를 생성하므로 메모리 낭비가 없습니다.  
노드의 삽입, 삭제가 포인터 변경만으로 가능하여 유연합니다.  

**단점:**    
부모 노드를 찾으려면 별도의 부모 포인터를 두거나 루트부터 다시 탐색해야 할 수도 있습니다.  
배열에 비해 포인터 저장을 위한 추가적인 메모리가 필요합니다.
