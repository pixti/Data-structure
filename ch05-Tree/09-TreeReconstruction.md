## 20. 이진 트리 복원 (Tree Reconstruction)
### 20-1. 순회 순서와 트리의 유일성
이진 트리를 순회하면 노드들의 일렬된 배열인 **순열(Permutation)** 이 생성됩니다.   
역으로 이 순열들을 이용해 원래의 이진 트리를 복원할 수 있는지 여부는 매우 중요한 문제입니다.  

* **전위 순열(Preorder Permutation)**: 루트 -> 왼쪽 -> 오른쪽 방문 순서  
* **중위 순열(Inorder Permutation)**: 왼쪽 -> 루트 -> 오른쪽 방문 순서  
* **후위 순열(Postorder Permutation)**: 왼쪽 -> 오른쪽 -> 루트 방문 순서  

### 20-2. 복원 가능 여부 판별  
하나의 순회 결과만으로는 원래 트리를 복원할 수 없습니다. 유일한 트리를 복원하기 위해서는   
반드시 **중위 순서(Inorder)** 를 포함한 두 개 이상의 순서 쌍이 필요합니다.  

1.  **전위(Pre) + 중위(In)**: **복원 가능**  
    * 전위 순서의 첫 노드가 항상 루트임을 이용해 중위 순서에서 왼쪽/오른쪽 서브트리 범위를 나눌 수 있습니다.  
2.  **후위(Post) + 중위(In)**: **복원 가능**  
    * 후위 순서의 마지막 노드가 항상 루트임을 이용해 서브트리 범위를 확정할 수 있습니다.  
3.  **전위(Pre) + 후위(Post)**: **복원 불가능 (유일하지 않음)**  
    * 루트는 알 수 있으나, 어느 노드가 왼쪽 자식인지 오른쪽 자식인지 구조적으로 확정할 수 없는 경우가 발생합니다.  


### 20-3. 복원 과정 예시 (Step-by-Step)

**입력**: 전위 순열 (ABCDEFGHI), 중위 순열 (BCAEDGHFI)  

1.  **루트 찾기**: 전위 순서의 첫 번째인 **A**가 전체 트리의 루트입니다.    
2.  **서브트리 분할**: 중위 순서에서 **A**를 기준으로 왼쪽은 **{B, C}**, 오른쪽은 **{E, D, G, H, F, I}** 로 나뉩니다.

    * 중위 순서에서 **B**의 왼쪽은 공백(null), 오른쪽은 **C**임이 확인됩니다.
