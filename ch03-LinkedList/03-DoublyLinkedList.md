# 3. 이중 연결 리스트 (Doubly Linked List, DLL)
단순 연결 리스트(SLL)의 `next` 포인터에 **`prev`(이전)** 포인터를 추가하여,   
양방향으로 자유롭게 소통할 수 있도록 개선된 자료구조입니다.

---
## 3-1. 이중 연결 리스트의 구조와 특징

### ① 노드(Node)의 구성
* **Next**: 뒤에 오는 노드를 가리키는 레퍼런스.
* **Prev**: 앞에 오는 노드를 가리키는 레퍼런스.
* **Item**: 실제 저장되는 데이터.

### ② 주요 장점과 단점
* **장점 1 (양방향 탐색)**: `Head`뿐만 아니라 `Tail`에서도 역방향으로 데이터를 찾아갈 수 있습니다.
* **장점 2 (삭제의 용이성)**: 삭제하려는 노드($p$)의 주소만 알면,  
  `p.prev`를 통해 이전 노드에 즉시 접근할 수 있어 **탐색 없는 삭제**가 가능합니다.
* **단점**: 노드마다 추가적인 포인터(`prev`)를 저장해야 하므로 메모리 사용량이 늘어나고,  
 연결 수정 시 관리해야 할 링크가 2배(앞뒤)가 되어 로직이 복잡해질 수 있습니다.


## 3-2. 구현의 핵심: 더미 노드 (Sentinel Node)
실제 구현에서는 예외 처리를 줄이기 위해 **더미 노드**가 있는 구조를 통상적으로 사용합니다.

* **Sentinel DLL**: 리스트의 양 끝에 데이터가 없는 가짜 노드인 **Header**와 **Trailer**를 세워둡니다.
* **효과**: 리스트가 비어 있어도 항상 "내 앞과 뒤에 누군가 존재한다"는 조건이 성립합니다.  
  덕분에 `if (head == null)` 같은 까다로운 예외 처리 없이 일관된 로직으로 삽입/삭제를 처리할 수 있습니다.


## 3-3. 삽입과 삭제 로직

### ① 노드 삽입 (Node Add)
노드 $p$ 와 그 다음 노드 사이에 새로운 노드 $q$ 를 삽입할 때는 총 **4개의 연결**을 수정해야 합니다. 
> **주의**: 연결이 끊어지는 사고를 막기 위해 **새 노드 ( $q$ )의 연결을 먼저 설정**하는 것이 안전합니다.

1. `q.prev = p;` (새 노드 앞 연결)
2. `q.next = p.next;` (새 노드 뒤 연결)
3. `p.next.prev = q;` (뒷 노드의 앞 연결 수정)
4. `p.next = q;` (앞 노드의 뒤 연결 수정)



### ② 노드 삭제 (Node Remove)
특정 노드 $p$ 를 삭제할 때는 $p$ 를 건너뛰도록 양옆 노드의 레퍼런스를 서로 이어줍니다.
1. **이전 노드 수정**: `$p.prev.next = p.next;$` ($p$ 의 이전 노드가 $p$ 의 다음을 가리킴)
2. **다음 노드 수정**: `$p.next.prev = p.prev;$` ($p$ 의 다음 노드가 $p$ 의 이전을 가리킴)


## 3-4. 이중 연결 리스트 (Doubly Linked List) - 시간 복잡도 정리

| 연산 유형 | 시간 복잡도 | 설명 |
| :--- | :---: | :--- |
| **맨 앞/맨 뒤 삽입·삭제** | $O(1)$ | Header와 Trailer 덕분에 탐색 없이 즉시 연산 |
| **위치를 모를 때 (탐색)** | $O(N)$ | 원하는 값을 찾기 위해 순차 탐색 필요 |
| **위치를 알고 있을 때** | $O(1)$ |  `prev` 덕분에 탐색 없이 즉시 연결 수정 가능 |


## 3-5. 이중 원형 연결 리스트 (Doubly Circular Linked List)
Header와 Trailer라는 두 개의 더미 노드를 **하나의 Dummy Head**로 통합하여 관리 효율을 극대화한 형태입니다.

### ① 왜 더미 노드 1개로 충분한가?
* **원형(Circular) 구조의 힘**: 
    - `dummy.next`는 첫 번째 실제 데이터 노드를 가리킵니다. (시작점)
    - `dummy.prev`는 마지막 실제 데이터 노드를 가리킵니다. (끝점)
* 더미 노드 하나가 리스트의 앞과 뒤를 동시에 연결해주는 가교 역할을 하므로 효율적입니다.

### ② 초기 상태 (Empty List)
* 데이터가 하나도 없을 때, `dummy.next`와 `dummy.prev`는 모두 **자기 자신(self)** 을 가리킵니다.
* 이 상태를 유지하면 첫 번째 노드가 삽입될 때  
  별도의 예외 처리 없이 "특정 노드의 앞/뒤"라는 동일한 규칙으로 매끄럽게 연결됩니다.

## 3-6. 이중 원형 연결 리스트 (Doubly Circular Linked List)의 시간 복잡도
이중 원형 연결 리스트는 이중 연결 리스트(DLL)의 모든 효율성을 계승하며, 단 하나의 더미 노드로 리스트의 양 끝단을 장악합니다.

| 연산 유형 | 시간 복잡도 | 설명 |
| :--- | :---: | :--- |
| **맨 앞 삽입·삭제** | $O(1)$ | `dummy.next`를 통해 즉시 접근 및 연산 |
| **맨 뒤 삽입·삭제** | $O(1)$ |  `dummy.prev`가 곧 마지막 노드이므로 탐색 없이 즉시 연산 |
| **위치를 알고 있을 때** | $O(1)$ | 내 앞(`prev`)과 뒤(`next`)를 바로 아니까 상수시간 소모 |
| **위치를 모를 때 (탐색)** | $O(N)$ | 인덱스가 없으므로 원하는 노드까지 순차 탐색 필수 |

 **완전한 양방향성**: 단방향 원형 리스트에서 불가능했던 '마지막 노드 삭제 후 Tail 이동'이   
 $O(1)$ 에 가능해졌습니다. (`tail.prev`로 바로 이동 하므로)
