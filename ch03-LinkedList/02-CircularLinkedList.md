# 2. 원형 연결 리스트 (Circular Linked List)
단일 연결 리스트의 마지막 노드가 `null` 대신 **첫 번째 노드(Head)**를 가리키도록 하여 고리(Loop) 형태로 만든 자료구조입니다.

## 2-1. 원형 리스트의 핵심 구조와 특징

### ① `tail.next` = `head`
* **구조적 혁신**: 별도의 `head` 포인터를 유지하지 않아도,  
  `tail` 변수 하나만 있으면 `tail.next`를 통해 $O(1)$ 시간 안에 머리(Head)에 접근할 수 있습니다.
* **장점**:   
    - 리스트의 시작과 끝에 즉시 접근이 가능합니다.  
    - 리스트 어느 곳에서도 중단 없이 순회가 가능합니다.  
    - 모든 노드가 `next`를 가지므로, 순회 시 `if (node == null)` 체크를 건너뛸 수 있는 경우가 많아 로직이 깔끔해집니다.  

### ② 단점과 주의사항  
* **무한 루프의 위험**: 리스트의 끝(`null`)이 없기 때문에,  
  제어 로직(탈출 조건)이 잘못되면 순회 시 영원히 루프를 도는 사고가 발생할 수 있습니다.
* **단방향의 한계**: 여전히 `next` 방향으로만 진행하므로, '뒤로 가기'는 불가능합니다.  
  (이 문제는 나중에 이중 연결 리스트에서 해결됩니다.)

## 2-2. 원형 리스트의 삽입 로직
### [Case A] 리스트가 비어있을 때 (tail == null)  
1. 새로운 노드 `newNode`를 생성합니다.  
2. **원형 유지**: 자기 자신을 가리키게 합니다. (`newNode.next = newNode`)  
3. `tail`이 이 노드를 가리키게 합니다.  

### [Case B] 리스트에 원소가 있을 때 (제일 앞/Head에 추가)    
1. **연결 고리 확보**: `newNode.next`가 현재의 머리(`tail.next`)를 가리키게 합니다.  
2. **꼬리와 연결**: 현재 꼬리(`tail`)의 다음이 `newNode`를 가리키게 수정합니다. (`tail.next = newNode`)  
3. **결과**: `tail.next`가 이제 `newNode`를 가리키므로, `newNode`가 새로운 머리(Head)가 됩니다.  


## 2-3. 원형 리스트의 삭제 로직 (Head 삭제)

### [Case A] 원소가 1개일 때  
* `tail.next == tail`이라면 노드가 하나뿐이라는 뜻입니다. 이때는 단순히 `tail = null`로 만들어 리스트를 비웁니다.  

### [Case B] 원소가 여러 개일 때  
1. **타겟 설정**: 삭제할 현재의 머리 노드를 `x`라고 정의합니다. (`x = tail.next`)  
2. **지름길 만들기**: `tail`이 `x`의 다음 노드를 바로 가리키도록 건너뛰게 합니다. (`tail.next = x.next`)  
3. **격리와 정리**: 삭제된 `x`의 `next`를 `null`로 만들어 연결을 완전히 끊어줍니다.  
   (이를 수행하지 않으면 참조가 남을 수 있어 GC가 수거하기 어려울 수 있으므로 명시적으로 끊어주는 것이 좋습니다.)


## 2-4. 시간 복잡도 분석

| 연산 유형 | 시간 복잡도 | 설명 |
| :--- | :---: | :--- |
| **삽입 (Head)** | **$O(1)$** | `tail.next`를 통해 즉시 삽입 |
| **삽입 (Tail)** | **$O(1)$** | Head 삽입과 로직은 같으나, 마지막에 `tail` 위치만 갱신 |
| **삭제 (Head)** | **$O(1)$** | `tail.next`만 변경하면 끝 |
| **삭제 (Tail)** | **$O(N)$** | **단점**: 삭제 후 새로운 `tail`이 될 '앞 노드'를 찾기 위해 처음부터 훑어야 함 |

### 왜 제일 뒤(Tail) 삭제는 여전히 $O(N)$인가?  
원형 리스트에서 `tail` 위치를 알고 있더라도, 그 노드를 삭제하면 **`tail` 바로 앞의 노드**가 새로운 `tail`이 되어야 합니다.   
하지만 단방향 리스트 특성상 내 앞이 누구인지 알려면 결국 `tail.next`(Head)부터   
다시 전체를 순회해야 하므로 $O(N)$ 의 시간이 소요됩니다.
